# Minedit Brand Assets

This first-pass identity direction uses a selected footprint plus edit cursor mark. It keeps the logo tied to the actual Minedit workflow: select two corners, describe the edit, and let the agent build inside the chosen area.

## Files

- `assets/minedit-mark.svg`: colored square mark for app-icon, README, and release-card use.
- `assets/minedit-mark-mono.svg`: single-color mark for badges, favicons, and UI chrome.
- `assets/minedit-logo.svg`: light-background horizontal lockup.
- `assets/minedit-logo-dark.svg`: dark-background horizontal lockup.

## Direction

- Preserve the `Minedit` spelling unless the project intentionally rebrands.
- Keep `minedit` lowercase for mod id, resource namespace, package paths, and config file names.
- Avoid Mojang, Minecraft, creeper-face, grass-block, or official game UI marks.
- Prefer the footprint/cursor mark over generic cube-only marks because it is more specific to Minedit's selection and editing workflow.

## Production Notes

The horizontal lockups use live SVG text for easy iteration. Before using the wordmark in a published app icon, store listing, or print asset, convert the wordmark to paths or replace it with a final typography system. The standalone mark is geometric SVG and is ready for small-size testing.
